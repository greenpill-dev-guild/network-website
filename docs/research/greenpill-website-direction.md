# Greenpill Website Direction

This is a strategy draft for the next version of the site, based on the current public footprint of Greenpill as of March 9, 2026.

## Positioning

Recommended top-line story:

Greenpill is a global regenerative network that helps local communities, builders, and storytellers coordinate real-world impact with Ethereum and web3 tools.

Why this works:

- it includes chapters
- it includes guilds
- it includes tooling
- it includes media
- it is understandable to non-Web3 visitors
- it does not trap the site inside `book brand` positioning

## Core audiences

| Audience | What they are really asking | What the site should prove | Primary CTA |
| --- | --- | --- | --- |
| Curious newcomer / Web2 visitor | "What is Greenpill and why should I care?" | Clear mission, real examples, accessible language | Learn the network |
| Local organizer / community builder | "Can I start or join something near me?" | Chapter structure, regional hubs, steward pathway | Join or start a chapter |
| Builder / product designer / engineer | "Is there real work here?" | Dev Guild, tools, builder spaces, open repos | Join the Dev Guild |
| Writer / educator / researcher | "Is there a place for my contribution?" | Writers Guild, publishing, sensemaking, topic pods | Join the Writers Guild / media work |
| Partner / funder / ecosystem org | "Is this credible, organized, and complementary?" | Programs, outcomes, partners, pilots, governance maturity | Partner with Greenpill |
| Existing community member | "Where do I plug in next?" | Current programs, events, calls, projects, updates | Find your next role |

## Narrative arc for the homepage

The homepage should take visitors through this sequence:

1. Greenpill is a living network.
2. The network runs through chapters, guilds, pods, and stewards.
3. The network produces visible work: local events, tools, media, and experiments.
4. Different people can enter through different doors.
5. There is a clear next step.

## Recommended information architecture

## 1. Home

Purpose:

- explain Greenpill fast
- show proof that the network is alive
- route visitors by intent

## 2. Chapters

Purpose:

- show active chapters and regional hubs
- tell local stories, not just list locations
- explain chapter onboarding and stewardship

Suggested content:

- featured chapter stories
- regional map or region-first index
- chapter demo day highlights
- start-a-chapter pathway

## 3. Guilds and Pods

Purpose:

- explain how skill-based and topic-based participation works
- make the Dev Guild and Writers Guild visible
- show topic pods such as Education, Environment, DeSci, and DePin

Suggested content:

- what each guild does
- current projects
- current working groups
- join flow

## 4. Projects and Protocols

Purpose:

- show what Greenpill is actually building and experimenting with
- make the Dev Guild output legible to partners and builders

Suggested content:

- Green Goods
- GreenWill
- Impact Reef
- Gardens
- Cookie Jar
- Allo / funding experiments
- impact reporting and reputation work

## 5. Learn

Purpose:

- keep books and podcast, but reposition them as part of the network learning stack

Suggested content:

- books
- podcast
- workshops
- onboarding flows such as Taking the Greenpill / Taking the DevPill
- chapter and guild resource kits

## 6. Stories

Purpose:

- highlight chapter updates, media, field reports, essays, and case studies
- create a bridge between website and Paragraph / YouTube / social

Suggested content:

- chapter stories
- guild updates
- Greenpill Garden updates
- podcast themes by topic
- partner stories

## 7. Partner / Fund

Purpose:

- make partnership, sponsorship, and collaborative research or implementation easier
- translate Greenpill for funders, ecosystem orgs, and public-goods operators

Suggested content:

- what Greenpill can do with partners
- areas of collaboration
- pilot examples
- why Greenpill is useful as connective tissue

## Homepage section concept

Recommended section order:

1. Hero
   Greenpill helps local communities, builders, and storytellers coordinate regenerative action.
2. How the network works
   Chapters x Guilds x Pods x Stewards x Tools
3. What is active now
   Community calls, Greenpill Garden, builder spaces, chapter updates
4. Featured chapter stories
   A few concrete local examples
5. What builders are shipping
   Green Goods, GreenWill, Impact Reef, Cookie Jar, etc.
6. Learning and media
   Books, podcast, workshops, publishing
7. Topic clusters
   Education, Environment, DeSci, DePin, Funding Mechanisms
8. Partner block
   Greenpill as a bridge organization
9. Get involved by role
   Newcomer, chapter steward, builder, writer, partner

## What to avoid in the redesign

- leading with a static slogan and leaving the real structure hidden
- making books the main frame again
- treating chapters as map pins without stories
- collapsing every contributor type into one generic `participate` page
- hiding current projects behind Discord or external tools
- assuming visitors already understand ReFi / public goods / Ethereum jargon

## Content priorities for the first serious refresh

If scope is limited, prioritize these first:

1. Rewrite the homepage narrative.
2. Add a real `Chapters` section with featured stories.
3. Add a real `Guilds / Pods` section.
4. Add a `Projects / Protocols` section for Dev Guild work.
5. Add a `Current Programs` section featuring Greenpill Garden and community calls.
6. Replace the old participation ladder with role-based pathways.

## Best steward-decision questions for tomorrow

- Which three chapters best show what Greenpill looks like on the ground?
- Which two guilds or pods are mature enough to present as flagship pathways?
- Which products should represent the builder side of the network?
- How much of the website should describe what Greenpill is versus what it is becoming?
- What is the one sentence a non-Web3 visitor should remember after leaving?
